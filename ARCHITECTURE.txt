RELEASE READINESS + FULL-SEASON VALIDATION V1 - ARCHITECTURE
===========================================================

1. RUNTIME CONFIGURATION
------------------------
Development imports:

src/environments/app-runtime.config.ts

Production builds replace that module with:

src/environments/app-runtime.config.production.ts

The production module is live-only and disables developer controls. The runtime
layer also checks the hostname, so an accidentally deployed development build
cannot use historical scoring on a production-like host unless it is explicitly
allowed in configuration.

2. FULL-SEASON SIMULATOR
------------------------
The simulator is deterministic and side-effect free. It makes no NHL requests
and no Firestore writes. It validates:

- four teams, 14 active roster slots, 3 flexible bench slots, and 3 IR slots per team;
- unique draft assets;
- 11 regular-season cycles;
- exact six-game slot windows;
- 7-vs-4 asynchronous advancement;
- missed appearances and zero-point appearances;
- queued roster moves and IR return boundaries;
- per-window projection immutability;
- live-scoring lease/cadence/write-suppression invariants;
- matchup completion and one-time standings application;
- four-team semifinals using Window 1;
- championship and third-place routing using Window 2;
- unique playoff bank assignments;
- first-through-fourth final placements and champion recording.

3. LIVE READINESS SNAPSHOT
--------------------------
Release Readiness performs on-demand commissioner reads of existing league
state. It does not create a new persistent readiness collection. It inspects:

- league and team configuration;
- draft status;
- latest and active cycles;
- projection metadata;
- global injury synchronization state;
- shared live-scoring control;
- current playoff state.

Checks are classified as pass, warning, or fail. Historical mode is a warning in
local development because the build is intentionally still in testing mode.

4. RECOVERY SAFETY
------------------
The stuck-lease recovery transaction refuses to clear a healthy unexpired lease.
Healthy workers must use the normal handoff flow. Only an expired lease or a
control document in error state can be cleared by the recovery button.

Projection regeneration uses the existing projection health checks. A collapsed
NHL statistics response cannot replace the last healthy projection board.

5. TEST UI LIFECYCLE
--------------------
Simulator logic remains useful as automated regression coverage. Production
builds hide the temporary UI controls now. The visual redesign cleanup will
remove obsolete standalone simulator pages/routes while retaining their pure
simulator functions and spec files.
