RELEASE READINESS + FULL-SEASON VALIDATION V1 - TEST PLAN
=========================================================

A. INSTALLATION SMOKE TEST
--------------------------
1. Stop and restart `npm start`.
2. Confirm Angular compiles successfully.
3. Confirm a historical-test banner appears below the navbar in local
   development.
4. Open League Hub -> More league tools -> Release Readiness.
5. Confirm the route loads only for the league commissioner.

B. FULL-SEASON SIMULATOR
------------------------
1. Select "Run Full Season".
2. Confirm 22 / 22 checks pass.
3. Confirm all eight milestones are marked passed:

   League and draft
   Regular season windows
   Roster transactions
   Projection freezing
   Shared live scoring
   Standings completion
   Banked playoffs
   Final placements

4. Expand the detailed checks and confirm no row is red.

C. LIVE READINESS CHECKLIST
---------------------------
1. Select "Refresh Checks".
2. In local development, confirm Historical scoring test mode is a warning.
3. Confirm six-game format passes.
4. Confirm scoring-rules and projection versions are displayed.
5. Confirm active-cycle schema checks pass when a cycle is active.
6. Confirm any scorer or injury error is surfaced as a failure rather than
   silently hidden.
7. Confirm playoff format version 2 passes when playoffs exist.

D. SAFE RECOVERY CONTROLS
-------------------------
Scoring refresh:
1. Select Request Score Refresh.
2. Confirm the diagnostics page later records the refresh.
3. Repeat without changing scores and confirm unchanged snapshots remain
   skipped.

Lease recovery:
1. While a healthy active lease exists, select Clear Stuck Lease.
2. Confirm the action refuses to interrupt the healthy worker.
3. Test again only with an expired or errored control document and confirm it
   clears safely.

Injury refresh:
1. Select Refresh Injuries.
2. Confirm the result message is shown.
3. Refresh readiness and confirm the latest successful sync is displayed.

Projection recovery:
1. Enter a valid target cycle.
2. Select Regenerate Projection.
3. Confirm the new snapshot id and target cycle appear in the success message.
4. Refresh readiness and confirm projection status is ready.

E. PRODUCTION SAFEGUARD CHECK
-----------------------------
On the user's Node v22.23.1 environment:

1. Run:

   npm run build

2. Serve the production output locally or deploy to a test host.
3. Confirm historical scoring is not active.
4. Confirm the historical-test banner is absent.
5. Confirm the following buttons are hidden:

   Run 7-vs-4 Simulator
   Test Banked Windows
   Run Scorer Tests
   Run Projection Tests

6. Confirm Release Readiness reports live NHL scoring configuration.

F. ACCESS CONTROL
-----------------
1. Sign in as a normal league manager.
2. Paste `/leagues/{leagueId}/release-readiness` directly.
3. Confirm the page shows Commissioner access required.
4. Confirm the normal manager can still use ordinary league, matchup, and team
   pages without regression.
