FANTASY HOCKEY — BANKED PLAYOFF WINDOWS V1
READY-TO-INSTALL PACKAGE

PURPOSE
This package lets every playoff roster slot continue into its next independent
six-game NHL window before the fantasy bracket destination is known. Once the
previous playoff matchup finishes, the already-started window is routed to the
Championship, 3rd Place, 5th Place, or consolation matchup.

REQUIRED BASELINE
Install this on top of the current project after the previously supplied:
- asynchronous cycle windows
- slot-boundary roster moves
- matchup window completion
- cycle transparency and simulator
- cycle simulator path fix
- projection-board recovery

PROJECT ROOT
/Users/StephenH/Documents/Programming/fantasy-hockey

INSTALL STEPS
1. Stop the Angular development server.
2. Back up the project or commit the current working state.
3. Open this package.
4. Copy its contents into the project root.
5. Allow all included source files and firestore.rules to replace the existing
   versions.
6. Deploy the included Firestore rules:

   firebase deploy --only firestore:rules --project nhl-fantasy-app-ab673

7. Restart the project with the known-good Node version:

   npm start

8. Hard-refresh the browser.
9. Open a league's Playoff Bracket page as commissioner.
10. Select "Test Banked Windows" and confirm all seven checks pass.

NEW FIRESTORE LOCATION
/leagues/{leagueId}/playoffWindowBanks/{ownerId}/windows/window-{N}

Each document stores one fantasy team's complete roster snapshot for one
postseason six-game window. It includes frozen projections, exact roster-slot
assignments, accumulated scoring progress, completion state, and eventual
bracket assignment.

IMPORTANT TEST-LEAGUE NOTE
Start a fresh playoff test after installing this package. A playoff that was
already actively scored under the old non-banked format did not preserve all
of the historical bank assignments needed for a perfect retroactive migration.
Regular-season data and leagues whose playoffs have not started are unaffected.

NO OTHER SETUP IS REQUIRED
