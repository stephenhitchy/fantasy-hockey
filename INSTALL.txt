Favorite-team persistence + Firestore rules fix V2

Replace these files in the project root:
- firestore.rules
- src/app/core/user/user.service.ts
- src/app/features/account/account-settings/account-settings.ts
- src/app/features/account/account-settings/account-settings.html
- src/app/features/account/account-settings/account-settings.css
- src/app/features/dashboard/dashboard.html
- src/app/features/dashboard/dashboard.css

Then run:
  npm run build

Deploy BOTH the rules and the hosted app:
  firebase deploy --only firestore:rules,hosting:app -m "Fix favorite team persistence"

Why the rules deployment matters:
- A manager can update only their own username/theme preference fields.
- A favorite-team-only write no longer fails because of an unrelated legacy username value.
- Signed-in matchup users can get a specific opponent profile by ID to read the favorite team.
- Listing the users collection remains blocked.
